# Mapping RBAC in BloodHound

**Speaker:** Hope Walker

## Abstract

This talk will discuss changes to BloodHound and how role-based access control (RBAC) is displayed in the graph. It will cover how RBAC has historically been mapped in attack paths and the shortfalls of this approach. I will then explain how to break roles down into discrete permissions and create edges based on those permissions rather than the role as a whole. Entra ID and Azure RM will be provided as examples of how to use this approach when mapping RBAC with OpenGraph.
